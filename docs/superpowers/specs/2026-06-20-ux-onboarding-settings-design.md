# Sqwizzey Switch — UX-проход: онбординг + визуальные настройки

Дата: 2026-06-20
Статус: дизайн утверждён, ожидает плана реализации

## Цель

Поднять UX двух поверхностей, не трогая саму карточку-оверлей:
1. **Первый запуск** — сейчас онбординга нет; новый юзер не понимает, зачем приложение и как настроить.
2. **Окно настроек** — выбор стиля/позиции текстовыми дропдаунами «вслепую»; превью статичное.

Ключевая идея — общие переиспользуемые компоненты: визуальный пикер стиля и анимированное превью живут и в визарде, и в настройках.

## Не в скоупе (явно)

- Вид самой карточки-оверлея и набор стилей — без изменений.
- Продуктовые «расширения идеи» (индикатор у курсора, Caps/Num Lock, профили, глобальный хоткей) — отложены, не в этой спеке.

## Часть A. Визард первого запуска

Отдельное окно `OnboardingWindow` (`ui:FluentWindow`, как `SettingsWindow`), показывается один раз.

**Триггер:** новый флаг `AppSettings.OnboardingDone` (default `false`). Если `false` при старте — App открывает визард вместо/перед обычным потоком; по завершении/пропуску ставим `true` и `Save()`.

**Шаги (3):**
1. **Привет + живое демо.** Логотип приложения, «Sqwizzey Switch», подзаголовок одной строкой. По центру — карточка языка, проигрываемая вживую (компонент `AnimatedPreview`). Без декоративных эмодзи. Кнопка «Поехали →».
2. **Сделай по-своему.** Компонент `StylePicker` (крупные карточки реального рендера) + сетка позиции 3×3 + `AnimatedPreview`, обновляется мгновенно. «← Назад» / «Далее →».
3. **Почти готово.** Тумблеры «Запускать вместе с Windows», «Показывать на активном окне». Мягкий нудж: одна строка про настройку Windows «свой язык на каждое окно» с кнопкой, открывающей `ms-settings:typing` (как в текущих настройках). Не усиливать. Кнопка «Готово ✓» → `OnboardingDone=true`, открыть настройки или просто закрыть.

**Пропуск:** ненавязчивая текстовая ссылка «Пропустить» внизу окна на всех шагах → ставит `OnboardingDone=true`, закрывает визард.

**Навигация:** простой индекс шага (0..2), показ/скрытие панелей шагов; лёгкая анимация перехода (fade/slide) опционально.

## Часть B. Визуальные пикеры в настройках

### B1. `StylePicker` (новый UserControl)
- Заменяет `CbStyle` (дропдаун) на сетку крупных карточек: по карточке на стиль (Frosted, Accent, Minimal, Pill, Neon).
- Каждая карточка рисует **реальный вид** стиля через `OverlayStyle.Apply(...)` в **настоящих пропорциях** (масштаб ~0.5): Pill 96×44, Minimal 92×66, Frosted/Accent/Neon 110×80; Accent = системный акцент. Это исключает рассинхрон с реальным оверлеем.
- Выбранная карточка — рамка-акцент (как тумблеры, `#A98BEC`).
- `SelectedStyle` (string) + событие `SelectionChanged`.

### B2. Позиция — сетка 3×3
- Заменяет `CbPosition` на сетку 3×3 (TopLeft…BottomRight + Center), маппинг на существующие значения `PositionMode`.
- **Условность:** секция «Позиция» + «Сдвиг X/Y» скрывается, когда включено «Показывать на активном окне» (follow-focus) — логика уже есть (`UpdatePositionVisibility` / `FollowFocus_Toggled`), просто применить к новой сетке. При follow-focus карточка летит в центр активного окна; позиция/сдвиг не применяются.

### B3. `AnimatedPreview` (новый UserControl)
- Сейчас превью статично (`PreviewCard`/`PreviewText` + `OverlayStyle.Apply`).
- Новое: **реально проигрывает** выбранную анимацию (скрэмбл текста, пружинное появление, и при желании каплю) тем же кодом-механизмом, что `OverlayWindow` (вынести анимации в общее место или вызвать аналогично).
- Авто-проигрывание при смене стиля/темы/анимаций; маленькая кнопка «▶» для повтора.

### Переиспользование
- `StylePicker` и `AnimatedPreview` — общие контролы; ставятся в `SettingsWindow` (раздел «Вид» и верх окна) и в шаг 2 визарда.

## Изменения данных/настроек

- `AppSettings`: добавить `bool OnboardingDone = false` (+ в `Clone()`, сериализуется как обычно).

## Затрагиваемые файлы (ориентир)

- Новое: `OnboardingWindow.xaml(.cs)`, `Controls/StylePicker.xaml(.cs)`, `Controls/AnimatedPreview.xaml(.cs)`.
- Правки: `App.xaml.cs` (триггер визарда по `OnboardingDone`), `SettingsWindow.xaml(.cs)` (замена `CbStyle`/`CbPosition` на пикеры, встраивание `AnimatedPreview`), `Models/AppSettings.cs`, `Localization.cs` (строки визарда + подписи).
- Возможный рефактор: вынести логику анимаций оверлея так, чтобы превью могло их проигрывать без дублирования.

## Локализация

Все новые строки — через `Loc`/`Localization.cs`, на 8 языков (как остальные).

## Верификация

- Сборка чистая (`dotnet build -c Release`).
- Скриншоты окон через временный CLI-хук + `PrintWindow(h,hdc,2)` (как делали в этой сессии): визард по шагам, настройки с новыми пикерами.
- Проверить: первый запуск открывает визард; `OnboardingDone` после завершения/пропуска = true и повторный старт визард не показывает; пикер стиля = реальный вид; позиция скрыта при follow-focus; превью проигрывает анимацию.
- Регресс: существующие стили/позиции/follow-focus/капля работают как раньше.

## Открытые мелочи (решить при реализации, дефолты заданы)

- Анимированное превью: авто-проигрывание при выборе + «▶» повтор (дефолт). 
- После «Готово» в визарде: открывать настройки или просто закрыть (дефолт — закрыть, оверлей уже работает).
